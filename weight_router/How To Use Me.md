# How To Use Me

This weight router selects the GuardianEye model checkpoint whose training
domain most closely resembles an uploaded video.

It chooses between:

- **HF:** hockey rink and hockey-broadcast footage.
- **NTU:** continuous CCTV/mobile footage and five-second NTU CCTV-Fights windows.
- **RLVS:** short, varied, web-sourced or handheld real-world clips.

The router also returns the selected model's correct violence-decision threshold.

## Important Limitation

The router works without access to the original training datasets. However,
until empirical profiles are generated from the exact training videos, the
reported similarity scores are heuristic estimates rather than calibrated
probabilities.

Domain similarity selects a model. It must not be treated as the model's
violence confidence.

## Required Folder Structure

Keep the entire `Model_Types` directory together:

```text
Model_Types/
|-- train_output_HF/
|-- train_output_NTU/
|-- train_output_RLVS/
`-- weight_router/
```

The router expects the three checkpoint folders to remain at these locations.

## Install Dependencies

Open PowerShell inside the router directory:

```powershell
cd D:\RAG_Graduation_Translator\Model_Types\weight_router
pip install -r requirements-video.txt
```

The video-analysis dependencies are:

- `numpy`
- `opencv-python`

## Run The Router

```powershell
python -m model_weight_router VIDEO.mp4 `
  --context "Description generated by the VLM" `
  --specifications '{"source":"cctv","camera":"fixed"}'
```

Example:

```powershell
python -m model_weight_router D:\videos\uploaded_clip.mp4 `
  --context "Continuous footage from a fixed security camera overlooking a street" `
  --specifications '{"source":"cctv","camera":"fixed","clip_structure":"continuous footage"}'
```

## Recommended Context And Specifications

Provide context that describes the video's domain and recording style. Useful
details include:

- Environment: `ice rink`, `arena`, `street`, `indoor`, or `public space`.
- Source: `broadcast`, `cctv`, `mobile`, `phone`, or `web video`.
- Camera: `fixed`, `wide angle`, `tracking`, `handheld`, or `moving`.
- Clip structure: `five second window`, `continuous footage`, `short event clip`,
  or `edited clip`.

Do not rely only on words such as `fight`, `violence`, or `assault`. These words
describe the classification task and do not identify the most suitable dataset.

## Output

The router returns JSON containing:

- `selected_dataset`: selected `HF`, `NTU`, or `RLVS` model.
- `selected_checkpoint`: full path to the selected checkpoint.
- `decision_threshold`: threshold to apply to the selected model's raw output.
- `calibrated_threshold`: available calibrated threshold.
- `routing_confidence`: confidence in the routing decision.
- `similarities`: ranked similarity estimates and selection reasons.

Example integration:

```python
from model_weight_router import ModelWeightRouter

router = ModelWeightRouter(r"D:\RAG_Graduation_Translator\Model_Types")

decision = router.route_video(
    r"D:\videos\uploaded_clip.mp4",
    context=vlm_overview,
    specifications={
        "source": "cctv",
        "camera": "fixed",
        "clip_structure": "continuous footage",
    },
)

classifier = load_classifier(decision.selected_checkpoint)
violence_probability = classifier(...)
verdict = violence_probability >= decision.decision_threshold
```

Only load the selected checkpoint into GPU memory.

## Validate The Installation

Run:

```powershell
python -m unittest discover -s tests -v
python -m model_weight_router --help
```

All routing tests should pass.

## Improve Similarity Later

When the exact training-video folders become available, generate measured
profiles from:

1. The exact HF training-video folder.
2. The exact RLVS training-video folder.
3. The generated NTU five-second training-window folder.

Run:

```powershell
python -m model_weight_router.profile_builder D:\path\to\training_videos `
  --output dataset_profile.json
```

Run this command separately for HF, NTU, and RLVS. The generated
`feature_profile` values can then replace the estimated values inside:

```text
weight_router/model_weight_router/profiles.json
```

For the strongest routing evaluation, also create a separate set of videos
labeled with which model should process each video. Use that set to tune and
measure routing accuracy.
